# Wiki Editorial Guidelines

> **Status:** Draft — wiki deployment is planned for Phase 2

## Overview

When the wiki at `wiki.breacher.net` launches, this document defines how content is created, reviewed, and published.

## Roles

| Role | Can Read | Can Edit | Can Publish | How to Get |
|------|----------|----------|-------------|------------|
| Public (anonymous) | ✅ | ❌ | ❌ | Anyone on the internet |
| Community Member | ✅ | ✅ (suggest edits) | ❌ | Discord OAuth login |
| Keeper | ✅ | ✅ | ✅ | Keeper role on Discord |
| Admin | ✅ | ✅ | ✅ | Authentik OIDC |

## Content Standards

- **Factual accuracy first** — Distinguish confirmed discoveries from theories
- **Evidence-based** — Link to sources (screenshots, timestamps, API data)
- **Neutral tone** — Document what happened, not what you think it means (save theories for the Theories section)
- **Keep it current** — Update pages when new information supersedes old

## Editorial Process

1. **Draft** — Anyone can suggest edits via the wiki editor
2. **Review** — Keepers review suggested edits for accuracy and tone
3. **Publish** — Keepers approve and publish changes
4. **Update** — Living documents; update as new information emerges

## Style Guide

- Use Markdown formatting
- Include dates for time-sensitive information
- Link to primary sources where possible
- Use the wiki's built-in image upload for screenshots
- Categorize content according to the wiki structure
